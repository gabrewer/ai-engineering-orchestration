# AI Engineering Orchestration

This is not a demo repository.

This is where I keep the agents, skills, and orchestration patterns I’m using to build systems with AI.

I’ve spent 30+ years writing software. What’s changing now isn’t just the tools — it’s how we work. More and more, building software is shifting from writing code to defining intent, constraints, and evaluating outcomes.

I don’t have this fully figured out. This repo is a way to share what I’m learning as I go.


---

## What This Is

A living collection of:

- Agents
- Skills
- Orchestration workflows
- Patterns and experiments

These are the actual artifacts I use (or am actively exploring) when building systems with AI.

This is not meant to be polished or complete. It will evolve.


---

## What’s Changing (My Current View)

AI is very good at reducing *accidental complexity* — the mechanics of writing code.

What it doesn’t remove is *essential complexity*:
- Understanding the problem
- Defining constraints
- Making tradeoffs
- Aligning stakeholders

That’s where most of the work is moving.

In my own workflow, I’m spending less time writing code directly, and more time:

- Defining intent
- Providing context and constraints
- Orchestrating multiple agents
- Evaluating outcomes and refining

That shift is what this repo is trying to make visible.


---

## How I’m Thinking About It

Right now, building software often looks less like:

> Write code → test → refactor

and more like:

> Define intent → provide constraints → generate → evaluate → refine → repeat

The tooling is evolving quickly, but the underlying pattern seems consistent:
- Less direct implementation
- More system-level thinking
- More emphasis on clarity and judgment
